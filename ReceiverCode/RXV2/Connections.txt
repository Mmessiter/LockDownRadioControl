I XIAO ESP32-S3 pad → goes to                  Radio 1     Radio 2     Radio 3
═══════════════════════════════════════════   ════════    ════════    ════════
 D2  (GPIO 4)   → CE  (chip enable)            CE pin       —           —
 D3  (GPIO 5)   → CSN (SPI chip select)        CSN pin      —           —
 D0  (GPIO 2)   → CE                            —          CE pin       —
 D1  (GPIO 3)   → CSN                           —          CSN pin      —
 D4  (GPIO 6)   → CE                            —           —          CE pin
 D9  (GPIO 9)   → CSN                           —           —          CSN pin

 D7  (GPIO 20)  → MISO  (shared bus)           MISO         MISO        MISO
 D8  (GPIO 8)   → SCK   (shared bus)           SCK          SCK         SCK
 D10 (GPIO 10)  → MOSI  (shared bus)           MOSI         MOSI        MOSI

 3V3 pad        → VCC   (shared)               VCC          VCC         VCC
 GND pad        → GND   (shared)               GND          GND         GND

 IRQ pin on every module → NOT CONNECTED      (we don't use the IRQ line)

Reserved for the flight controller — leave clear:
- D5 (GPIO 7) → UART RX from FC (CRSF telemetry in)
- D6 (GPIO 21) → UART/RMT TX to FC (SBUS / CRSF / IBUS / PPM out)

Reminders:
- This is an S3-only layout — D9 is a regular GPIO on S3, but on the older C3 it's a BOOT-strap pin and Radio 3 would be unreliable
- All three sockets sharing SCK/MISO/MOSI is fine — only one radio has CSN low at a time, so only one talks
- ML01SP4 / DP5 pinouts: GND, VCC, CE, CSN, SCK, MOSI, MISO, IRQ (in that order on the 2×4 header). Wire each socket the same; only the CE/CSN pair varies
- Keep antenna ends spaced as far apart as practical — ideally orthogonal to each other for diversity

With pluggable sockets you can test every combination (just 1 in slot 1, just 1 in slot 3, slots 1+2, slots 2+3, all three, etc.) and the firmware will adapt automatically — the home page banner tells you which slots came up at each boot.
